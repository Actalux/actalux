# City of Clayton budget ingest — methodology

How the City of Clayton budget figures on `/mo/clayton/council/budget` are
sourced, extracted, and cited. Every figure traces to the city's audited
financial statements; nothing is estimated or imputed.

## Source

The figures come from the **City of Clayton Annual Comprehensive Financial
Reports (ACFRs)**, fiscal years ended September 30, 2020 through 2024, as filed
with the **Missouri State Auditor's** political-subdivision repository
(`auditor.mo.gov`). Clayton is polysub "City of Clayton" (09-096-0018) in St.
Louis County (096). The State Auditor holds each city's full audited statements
as a public PDF, on an open host; the city's own site (`claytonmo.gov`) carries
the same documents but sits behind an Akamai bot-block, so the State Auditor copy
is the authoritative, retrievable source. `scripts/fetch_city_acfr.py` downloads
the five audited ACFRs (the full financial-statement PDFs, not the short single-
audit reports) and records the file URL for each year as provenance.

## What is loaded

For each year, read from the audited **Statement of Revenues, Expenditures and
Changes in Fund Balances — Governmental Funds**:

- **By fund** — total revenues, total expenditures, and ending fund balance for
  each governmental fund column.
- **Revenue by source** — the Total Governmental Funds figure for each revenue
  line (property taxes, sales taxes, intergovernmental, etc.).
- **Expenditure by function** — the Total Governmental Funds figure for each
  function line (general government, public safety, public works, etc.).

And from the Required Supplementary Information:

- **General Fund budget vs. actual** — total revenues and total expenditures at
  the original budget, final budget, and actual, from the General Fund budgetary
  comparison schedule.

Clayton has only governmental and fiduciary funds — no enterprise/proprietary
funds — so the governmental-funds statement covers the operating budget in full.

## Two caveats recorded with the figures

- **The major-fund set changes year to year.** Six fund columns in FY2020–21
  (a Capital Projects fund and a 2014 General Obligation Bond fund were then
  major), five in FY2022, four in FY2023–24. The loader reads each year's actual
  columns from the data rather than assuming a fixed set.
- **The General Fund budget schedule's basis differs by year.** FY2020 and
  FY2021 state a "budget basis"; FY2022–FY2024 state a basis generally
  consistent with GAAP. The exact wording each ACFR uses is stored in every
  budget-vs-actual figure's note, so the citation reflects what the document
  itself says.

The city fiscal year ends September 30, so FY2024 spans Oct 1, 2023 – Sep 30,
2024 and is stored as `fiscal_year = "2023-2024"` to match the chart's span
format.

## How figures are cited

Each year's figures are rendered into a clean markdown digest — one verbatim,
self-describing row per figure — which is ingested as a cited document under the
City Council entity (`source_portal = "auditor"`). The raw audit tables are not
used as the citation surface: under PDF extraction their row labels decouple from
their values, so a row-level citation against the raw table is unreliable. The
digest is the citation surface; the original audited PDF is the verifiable
record. `source_url` points to the State Auditor file, and the ACFR PDF is
uploaded to storage so the reader's "Open original" embeds the real audit. Each
figure carries the stable `citation_id` of the digest chunk it was read from.

## Integrity guards

Reconciliation is enforced twice — at extraction and again at load
(`load_city_budget.py`, asserted before any row is written):

- per-fund revenues, expenditures, and ending balances each sum to the audit's
  stated Total Governmental Funds column;
- revenue-by-source sums to total revenues;
- expenditure-by-function sums to total expenditures.

A transcription or parsing error fails the load loudly rather than publishing a
wrong figure. The current load is **212 figures across five years** (FY2020–24),
every one citing a source chunk.

## Reproduce

```
uv run python scripts/fetch_city_acfr.py
doppler run --project mac --config dev -- uv run python scripts/load_city_budget.py --dry-run
doppler run --project mac --config dev -- uv run python scripts/load_city_budget.py
```

The verified figures of record are committed at `scripts/city_budget_figures.json`
(amounts in whole dollars, with each year's statement/RSI page labels). The
downloaded ACFR PDFs live under `data/city_acfr/` (gitignored; regenerated by
the fetch script).
